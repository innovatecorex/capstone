F1 — ENCRYPT LRN AT REST (with searchable hash)  ** TEST LOCAL FIRST **
=======================================================================

WHAT THIS DOES
  Encrypts the student LRN at rest (AES-256, via Crypt) exactly like email,
  with an lrn_hash (SHA-256) column for equality lookups, uniqueness, and
  search. Raw DB now shows ciphertext for LRN — matching your Chapter 3 claim.

IMPORTANT BEHAVIOR CHANGE
  LRN search is now EXACT (by hash), not partial. Typing a full LRN finds the
  student; partial LRN no longer matches (you cannot substring-search encrypted
  data). Name/username search is unchanged. Applicant-table LRN search (a
  different table, not encrypted) is intentionally left as partial.

  Defense line: "LRN is AES-256 encrypted at rest using the same searchable-hash
  pattern as email; equality and duplicate checks run against the SHA-256 hash,
  never plaintext — so partial-text search on LRN is intentionally unavailable,
  exactly as you couldn't substring-search a hashed password."
